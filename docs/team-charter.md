## Working Agreements
- **Code Review SLA:**
  - Code reviews completed within 24 hours of PR submission
- **Branch Naming Convention:**
  - `feature/<short-description>`
  - `bugfix/<short-description>`
  - `hotfix/<short-description>`
- **Commit Format:**
  - `<type>: <short description>`
    - Examples: `feat: add login page`, `fix: correct API endpoint`

  1. Lucia
  2. Nick

## Conflict Resolution Process
1. Discuss the issue directly between those involved.
2. If unresolved, bring the issue to the next team meeting for group discussion.
3. If the conflict persists or impacts progress, escalate to faculty supervisor or external facilitator.
4. Always maintain respect, listen actively, and aim for consensus.
